# Prompt copy-paste để triển khai HelpID

Ghi chú vận hành:

- File này thay thế `harness-engineering/prompts-dang-ky-dang-nhap.txt`; từ bây giờ dùng `harness-engineering/copy-paste-prompts.txt` cho các prompt triển khai copy-paste.
- Copy từng prompt theo thứ tự, không chạy nhiều prompt cùng lúc.
- Mỗi prompt đều yêu cầu agent đọc `AGENTS.md` và `harness-engineering/ke-hoach.md` trước khi sửa.
- Sau prompt nào có sửa code/tài liệu/cấu hình, agent phải cập nhật `CHANGELOG.md` theo rule repo.
- Nếu prompt làm thay đổi use-case, phải cập nhật `harness-engineering/uml-use-case.puml`, xóa `harness-engineering/uml-use-case.png` cũ, rồi render lại PNG mới.
- Quy tắc “frontend đúng” chỉ áp dụng khi frontend đã bám đúng contract đã chốt. Với bảo mật, backend vẫn phải tự kiểm tra auth/permission, không tin dữ liệu client gửi lên.

NHÓM 10: CERTIFICATE PINNING — CHẶN MITM ATTACK
============================================================

Bối cảnh: Android app hiện giao tiếp với backend ASP.NET Core qua HTTP plaintext (`http://10.0.2.2:5080` trên emulator, cấu hình trong `HelpIdApiConfig.kt`). JWT access token, refresh token, và password được gửi qua wire không mã hóa — kẻ tấn công cùng mạng WiFi có thể dùng proxy Charles / Burp Suite / mitmproxy để đọc toàn bộ credential trong transit. Certificate Pinning là lớp bảo vệ buộc OkHttp từ chối kết nối đến bất kỳ server nào không có TLS certificate public key khớp với hash đã pin — kể cả khi kẻ tấn công đã cài root CA giả vào hệ thống và thực hiện SSL interception thành công. Triển khai gồm 2 phần: (1) bật HTTPS trên backend dev và extract SPKI SHA-256 fingerprint; (2) cài CertificatePinner vào OkHttp client duy nhất được centralize.

--- PROMPT 52: Chốt thiết kế Certificate Pinning và phân tích codebase trước khi code ---

production dùng CA thật nên không cần override; (8) **Pin failure UX** — không crash app; không fallback HTTP (tạo false sense of security); hiển thị error "Không thể kết nối an toàn với máy chủ" kèm nút Retry;

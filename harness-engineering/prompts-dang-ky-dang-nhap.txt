# Prompt copy-paste để triển khai đăng ký/đăng nhập HelpID

Ghi chú vận hành:

- Copy từng prompt theo thứ tự, không chạy nhiều prompt cùng lúc.
- Mỗi prompt đều yêu cầu agent đọc `AGENTS.md` và `harness-engineering/ke-hoach.md` trước khi sửa.
- Sau prompt nào có sửa code/tài liệu/cấu hình, agent phải cập nhật `CHANGELOG.md` theo rule repo.
- Nếu prompt làm thay đổi use-case, phải cập nhật `harness-engineering/uml-use-case.puml`, xóa `harness-engineering/uml-use-case.png` cũ, rồi render lại PNG mới.
- Quy tắc “frontend đúng” chỉ áp dụng khi frontend đã bám đúng contract đã chốt. Với bảo mật, backend vẫn phải tự kiểm tra auth/permission, không tin dữ liệu client gửi lên.
